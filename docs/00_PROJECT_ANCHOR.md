# 00 PROJECT ANCHOR

## Core decision

Dinagat Pass is a standalone app and standalone repository.

It must not be built as an OSP module, OSP extension, OSP sub-app, Nomad Academy module, or CABrainworks module. - reference-only anti-drift boundary; not a parent app, not shared runtime, and not a source of Dinagat Pass authority.

Correct relationship:

- Dinagat Pass = separate product
- One Siargao Pass = architectural reference only - reference-only anti-drift boundary; not a parent app, not shared runtime, and not a source of Dinagat Pass authority.

## Product thesis

Dinagat Pass is not a generic tourism app.

It is a Destination Economy Operating System for Dinagat Islands.

It coordinates:

- traveler identity
- official QR/pass
- trip records
- destination discovery
- operator fulfillment
- booking requests
- operator confirmation
- payment readiness
- site access points
- QR scan events
- visit stamps
- LGU/admin governance
- sustainability controls
- source attribution
- audit logs

## Operating truth doctrine

- QR / Scan Event = operational truth
- Visit Stamp = derived engagement output
- Booking = request / commercial lifecycle
- Payment Readiness = backend-controlled commercial gate
- Operator Capability = fulfillment authority
- Site Access Point = location/action context
- Source Attribution = economic accountability
- Marketplace Exposure = governed public visibility
- Audit Event = governance accountability

## Hard interpretation

- Request is not confirmation.
- Accepted is not confirmed.
- Paid is not completed.
- Stamp is not source truth.
- Exposure score is not approval.
- Operator approval is not capability approval.
- Discovery is not permission.
- Visibility is not bookability.
- Bookability is not payment readiness.
- QR validation is operating truth.

## MVP proof chain

Traveler discovers Dinagat experience
-> sees official route/product
-> sees verified local partner supply
-> submits request
-> operator receives request
-> operator accepts/confirms
-> payment/request status becomes ready
-> booking attaches to Dinagat Pass
-> QR validation is possible
-> Visit Stamp can be issued from governed validation
-> LGU/province sees activity

If this chain is not proven, the app is UI theater.
