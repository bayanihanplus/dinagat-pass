# 02 NUCLEAR UI / UX DOCTRINE

Dinagat Pass must be built on a world-class global trust UI/UX architecture.

It must not look like a generic tourism app, generic SaaS dashboard, generic travel blog, or template marketplace.

## Required interface qualities

- official
- premium
- compact
- intuitive
- direct
- mobile-first
- QR-first
- status-aware
- operator-protective
- LGU-trustworthy
- eco-sensitive
- global-standard

## Hard rejections

- no generic UI
- no card dump
- no dominant green
- no heavy dark hero blocks
- no bulky typography
- no weak CTAs
- no inconsistent border radius
- no random colors
- no long-copy traveler cards
- no fake scan success
- no payment CTA before backend readiness
- no stamp UX before QR truth
- no template dashboard look
- no uncontrolled gradients
- no generic bootstrap blue
- no paragraph-heavy traveler surfaces

## Approved visual direction

- Deep Island Navy = trust, official labels, primary text
- Ocean Teal = primary actions, QR actions, active states
- Sun Gold = verified accents and confirmation highlights
- White = card surfaces
- Soft Aqua Mist = app backgrounds and shells
- Soft Slate = secondary text

## Behavioral doctrine

Payment UX must follow backend readiness.

Stamp UX must follow QR / Scan Event truth.

Booking UX must show request and commercial lifecycle honestly.

Marketplace exposure must be governed public visibility, not random listing.

Operator capability must control fulfillment authority.

Source attribution must preserve economic accountability.

Audit events must support governance accountability.
