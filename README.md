# Dinagat Pass

Dinagat Pass is a standalone app and standalone repository.

It is not an extension, module, sub-app, fork, or child surface of One Siargao Pass, Nomad Academy, or CABrainworks.

One Siargao Pass is architectural reference only. Dinagat Pass may mirror selected operating logic from OSP, but it must remain a separate product, separate codebase, separate deployment target, and separate governance system.

## Core thesis

Dinagat Pass is a Destination Economy Operating System for Dinagat Islands.

It coordinates traveler identity, official QR/pass behavior, discovery, booking requests, operator fulfillment, payment readiness, site access, QR scan events, visit stamps, marketplace exposure, source attribution, and governance auditability.

## Operating truth doctrine

- QR / Scan Event = operational truth
- Visit Stamp = derived output
- Booking = request / commercial lifecycle
- Payment Readiness = backend-controlled
- Operator Capability = fulfillment authority
- Site Access Point = location/action context
- Source Attribution = economic accountability
- Marketplace Exposure = governed public visibility
- Audit Event = governance accountability

## Build status

This repository currently contains only the standalone repo shell and doctrine/design-system foundation.

No frontend pages, backend services, database schema, auth implementation, or deployment configuration have been scaffolded yet.
